# Star Frontier（星海边境）

浏览器端 3D 开放宇宙飞行射击游戏。技术栈：Three.js + 原生 ES6 模块，无打包器、无后端。

## 当前状态：Phase 8 完成（动态战争 / 随机事件 / 舰队AI）

在 Phase 1~7（飞行框架、战斗、银河生成与行星降落、NPC/空间站/经济、阵营与任务/声望、模块化改装、存档与UI）的基础上，本次交付实现了需求文档「八、动态银河」「十、动态战争」「十七、随机事件」要求的核心——银河不是静止的，玩家的行动真的会改变局势。

**动态战争（`faction/GalacticWarSimulator.js`）**
- 每个阵营有一个"实力"数值：玩家消灭海盗会削弱海盗、增强三大正规阵营；完成任务会增强对应阵营的实力——这是玩家行动影响战局的具体机制，不是装饰
- 只跟踪玩家实际到访过的扇区（银河大概率是无限的，为没去过的地方维护战争状态没有意义），每隔一段时间按"当前控制阵营 vs 挑战阵营"的实力对比滚一次易主判定，概率压得很低（单次 ≤15%），银河局势是"过一阵子回头看，发现变了"的量级
- 易主结果实时接入空间站归属/HUD星域显示/任务板/市场折扣——不是只在后台跑着自己玩的空气数值，而是真正影响你在对应站点能接到什么任务、买卖打几折
- 完整持久化进 Phase7 的存档系统，读档后战争局势不会重置

**随机事件（`galaxy/RandomEventSystem.js`）**
- 空间风暴（机动性下降）、海盗突袭（额外一波敌人逼近）、残骸发现（信用点奖励）——先做这三种能用现有系统稳健实现、玩家能立刻感知到差异的类型，而不是塞一堆只有提示文案没有机制效果的"装饰性事件"

**舰队AI**：`enemy/EnemyAIController.js` 的交战盘旋角度从"非左即右"两个选项升级为任意角度，编队出生的敌人会被分配到均匀分布的不同角度，围攻玩家时自然从不同方向包抄——海盗突袭事件与日常补给都有概率成队而不是清一色独狼。

**顺手修的一个问题**：敌人生成点一直是相对世界原点 `(0,0,0)` 计算的，Phase2 阶段没问题（玩家就在原点附近），但 Phase3 银河流式加载上线后，玩家飞远了敌人还在原点附近刷，变成"刷在玩家去不了的地方"。这次改成相对玩家当前位置计算，海盗突袭事件也因此能正确出现在玩家身边。

## 如何运行（重要）

飞船控制脚本使用原生 ES6 `import/export` 模块。**主流浏览器出于安全策略，禁止通过 `file://` 协议直接加载本地 ES 模块**，因此不能简单双击 `index.html` 打开——这不是本项目的 bug，而是浏览器的模块安全限制，所有基于 ES Module 的多文件网页项目都会遇到。需要用任意一种方式起一个本地静态服务器，例如：

```bash
# 方式一：Python（大多数系统自带）
cd StarFrontier
python3 -m http.server 8080
# 然后浏览器打开 http://localhost:8080

# 方式二：Node.js
npx serve StarFrontier -l 8080

# 方式三：VS Code 的 "Live Server" 插件，右键 index.html -> Open with Live Server
```

操作方式：`W/S` 俯仰，`A/D` 偏航，`Q/E` 横滚，`↑/↓` 油门，`Shift` 冲刺，`Space` 刹车，**鼠标左键** 开火主武器，**F** 发射追踪导弹，**L** 降落/起飞，**G** 地表采矿，**K** 空间站对接/解除对接，**Esc** 暂停。

## 关于本阶段范围的诚实说明

- **动态战争是"实力驱动的概率状态机"，不是真正的舰队调度/战场模拟**。银河这么大，真的去模拟每个阵营的舰队在哪打仗、怎么移动，投入产出比对现在这个阶段来说太低。但概率背后的实力数值是玩家真实行动喂出来的，这条因果链是真的：多打海盗、多接某个阵营的任务，那个阵营在后续易主判定里就真的更占优势。
- **只实现了 3 种随机事件**，不是需求文档列出的全部类型（超新星/黑洞/虫洞/AI入侵/海盗大会……）。这些类型大多需要对应的天体/敌人类型先存在（比如"AI入侵"需要 AI 文明这个隐藏阵营真正落地），硬塞一个只有 toast 文案、没有机制效果的"装饰性事件"不如不做。
- **舰队AI 只做了"出生在一起 + 包抄角度错开"**，没有真正的编队跟随飞行（僚机时刻保持相对领队的队形）或领队阵亡后的重新编组。这两者需要更复杂的僚机状态机，而"围攻时自然散开成合围之势"已经能让玩家明显感觉到"这是一队敌人在配合"，是当前投入产出比更高的取舍。
- **战斗 AI 的敌对判定仍然是静态的**——声望降到很低也不会让巡逻舰主动打你（因为压根没有"巡逻舰"这个 archetype，敌人只有海盗一种）。真正的多阵营战斗需要给三大正规阵营也做出可战斗的舰船类型，属于比"银河局势会变"更大的工程量，本阶段没有强行拼凑。
